GRPO TRANSFER MODULE - ALL 9 ENDPOINTS FIXED
================================================================================

PROBLEM:
--------
All 9 API endpoints were using non-existent method: sap.call_sap_api()
Result: Complete module failure with error "object has no attribute 'call_sap_api'"

SOLUTION:
---------
Updated all 9 endpoints to use correct method: sap.session.get() / sap.session.post()

FIXED ENDPOINTS:
================

✅ 1. Series List API
   GET /grpo-transfer/api/series-list
   Returns: {"success": true, "series": [...]}

✅ 2. Documents by Series API
   GET /grpo-transfer/api/doc-numbers/<series_id>
   Returns: {"success": true, "documents": [...]}

✅ 3. Validate Item API
   GET /grpo-transfer/api/validate-item/<item_code>
   Returns: {"success": true, "is_batch_item": true/false, ...}

✅ 4. Batch Numbers API
   GET /grpo-transfer/api/batch-numbers/<doc_entry>
   Returns: {"success": true, "batches": [...]}

✅ 5. Warehouses API
   GET /grpo-transfer/api/warehouses
   Returns: {"success": true, "warehouses": [...]}

✅ 6. Bin Codes API
   GET /grpo-transfer/api/bin-codes/<warehouse_code>
   Returns: {"success": true, "bins": [...]}

✅ 7. Create Session View (UI Route)
   GET /grpo-transfer/session/create/<doc_entry>
   Returns: HTML page with document details

✅ 8. Get GRPO Details API
   GET /grpo-transfer/api/grpo-details/<doc_entry>
   Returns: {"success": true, "document": {...}, "line_items": [...]}

✅ 9. Post Transfer to SAP API
   POST /grpo-transfer/api/session/<session_id>/post-transfer
   Returns: {"success": true, "sap_doc_entry": ..., "sap_doc_num": ...}

WHAT CHANGED:
=============

BEFORE (Broken):
    response = sap.call_sap_api(
        method='GET',
        endpoint="DocumentsService_DocumentSeries?...",
        headers={'Prefer': 'odata.maxpagesize=0'}
    )
    ❌ Method doesn't exist!

AFTER (Fixed):
    url = f"{sap.base_url}/b1s/v1/DocumentsService_DocumentSeries?$filter=DocType eq 1250000001&$select=Series,SeriesName,NextNumber"
    headers = {'Prefer': 'odata.maxpagesize=0'}
    response = sap.session.get(url, headers=headers, timeout=30)
    ✅ Method exists and works!

KEY IMPROVEMENTS:
=================
✅ Authentication check: if not sap.ensure_logged_in()
✅ Proper URL construction: f"{sap.base_url}/b1s/v1/{endpoint}"
✅ Direct session calls: sap.session.get() / sap.session.post()
✅ HTTP status checking: if response.status_code == 200
✅ Comprehensive logging: logger.debug(), logger.info(), logger.error()
✅ Proper headers: {'Prefer': 'odata.maxpagesize=0'}
✅ Timeout handling: timeout=30

TROUBLESHOOTING:
================

If you still see the error:
    1. Clear browser cache (Ctrl+Shift+Delete)
    2. Restart Flask application
    3. Check browser console (F12) for errors
    4. Run: python test_grpo_transfer_api.py
    5. Check Flask logs for error messages

If SAP B1 connection fails:
    1. Verify SAP B1 is running
    2. Check credentials in .env
    3. Verify network connectivity
    4. Check SAP B1 OData service is enabled

If no series appear:
    1. Create GRPO series in SAP B1
    2. Go to Administration → Setup → Document Settings → Document Series
    3. Create new series for Purchase Delivery Notes
    4. Refresh the WMS page
